# 多子女課程管理功能 - MVP 精簡方案

## 🎯 功能目標

讓家長在自然語言中包含子女名稱來管理課程，例如：「小明明天下午兩點鋼琴課」

## 🚀 核心設計決策

### 第一性原則分析
**真實需求**: 區分不同子女的課程
**最簡解法**: 在課程名稱中嵌入子女信息

### 語義嵌入法（推薦方案）
```yaml
輸入: "小明明天下午兩點鋼琴課"
處理: course_name = "小明鋼琴課"
存儲: 使用現有 courses 結構，無需修改
```

## ⚡ 方案優勢

1. **零架構改動** - 完全兼容現有系統
2. **實現極簡** - 僅需修改語義處理邏輯
3. **開發快速** - 預計 4 小時完成
4. **風險極低** - 不影響現有功能

## 📊 方案對比

| 方案 | 開發時間 | 架構影響 | 代碼量 | 風險 |
|------|---------|---------|--------|------|
| **語義嵌入法** | 4小時 | 零 | ~100行 | 極低 |
| 複雜實體系統 | 7天 | 重大 | 2500+行 | 高 |

## 🎯 測試目標

- ✅ "小明明天下午兩點鋼琴課" → 課程名稱顯示「小明鋼琴課」
- ✅ "小美每週三早上十點法語課" → 課程名稱顯示「小美法語課」
